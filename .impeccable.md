# TrueScore

A browser extension that replaces inflated 4.2-vs-4.3 star ratings with scores that actually mean something, across shopping, travel, and entertainment sites.

## Design Context

### Users
Savvy consumers who don't trust star averages. They're on Amazon / Booking / Google Maps / Letterboxd / Goodreads in a decision-making moment — about to buy, book, watch, or visit — and want the truth behind the rating in seconds. They are skeptical, analytical, and will notice if a number looks decorative instead of earned.

### Brand Personality
Analytical, editorial, confident. Three words: **precise, discerning, quiet**. The interface should feel like a Bloomberg terminal crossed with a well-designed magazine sidebar — numbers as the protagonist, typography as the craft, decoration nowhere. No marketing sheen. Evokes trust through restraint.

### Aesthetic Direction
Dark glass islands embedded into host sites. Panels float or dock with a clear identity of their own — TrueScore never pretends to be part of the site it's augmenting.

- **Type**: `IBM Plex Mono` 500/600 for all numerals, scores, counts, counters. `Sora` 300–600 for UI text. Micro-labels in 9–10px, uppercase, 0.08–0.12em tracking.
- **Surface**: `rgba(14, 16, 24, 0.9)` with `backdrop-filter: blur(24px) saturate(180%)`, hairline `rgba(255,255,255,0.07)` border, 12–16px radius. Avoid pure black.
- **Accent**: warm gold `#E8B86D` → cool teal `#6DD3CE` gradient — reserved for the brand signature (top hairline, primary bars, value-for-money stars). Do not overuse.
- **Sentiment**: `#4ADE80` positive, `#F87171` negative, amber `#E8B86D` middle. Tint, don't saturate.
- **Neutrals**: `rgba(255,255,255, 0.88 / 0.7 / 0.5 / 0.3 / 0.06)` — a strict ladder for hierarchy.
- **Motion**: `cubic-bezier(0.16, 1, 0.3, 1)` for entrances; width transitions 400–600ms; subtle opacity pulses for loading. No bounce. No elastic.
- **Layout**: asymmetric, left-aligned, generous negative space. Data rows are tabular-num and right-aligned on numerics. Numbers are always the biggest thing on screen.

**Anti-references**: glassmorphism-for-decoration, purple-to-blue AI gradients, icon-above-heading card grids, rounded-with-colored-border accents, sparklines as garnish, bounce easing, gray-on-colored text.

### Design Principles
1. **Numbers lead, chrome follows.** The metric is the hero. Labels whisper, digits speak.
2. **Tabular is non-negotiable.** Monospaced, right-aligned, tabular-nums on every count and percent. Columns must align between rows.
3. **Micro-label grammar.** 9–10px, uppercase, `0.08–0.12em` letter-spacing, 40–50% white. Every section is introduced this way — never with a H2.
4. **One gold-teal signature per panel.** The gradient is a signature, not a style. Use it once, meaningfully (a hairline, a primary bar). Multiple uses cheapen it.
5. **Earned hierarchy.** Every opacity step in the neutral ladder must mean something. If two elements share `0.5`, they're peers; if one is `0.3`, it's secondary by design, not by accident.
6. **Restraint over decoration.** No icons next to headings. No gradient text. No decorative shadows. Any detail that doesn't encode information is removed.
